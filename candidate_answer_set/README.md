# Candidate Answer Set Generation

This repository provides code for generating and analyzing candidate answer sets for both **factual (MCQ)** and **reasoning** datasets. It also supports answer recovery and dataset preparation for probing-based evaluation.


## 📌 Overview

We provide scripts for:

* Generating candidate answer sets from LLMs
* Recovering answers from reasoning outputs
* Preparing datasets for probing-based evaluation

The pipeline supports both **behavioral metrics** and **internal (probing-based) metrics**.


## ⚙️ Installation

```bash
conda env create -f environment.yml
conda activate pkex
```


## 🚀 Pipeline Workflow

The overall workflow is as follows:

1. Generate candidate answer sets

   * `ca_mcq.py` for factual (MCQ) datasets
   * `ca_reasoning.py` for reasoning datasets

2. Recover answers (for reasoning datasets)

   * `recover_answer.py`

3. Split dataset for probing

   * `split_for_probing.py`


## ⚡ Quick Start

### Step 1: Generate candidate answers (MCQ)

```bash
CUDA_VISIBLE_DEVICES=0 python ca_mcq.py \
  --model_size 7B \
  --input_jsonl /mnt/PK/MCQ/final/AH_rebuttal_new.jsonl \
  --prompt_id 1
```


### Step 2: Generate candidate answers (Reasoning)

```bash
CUDA_VISIBLE_DEVICES=0 python ca_reasoning.py \
  --model_size 7B \
  --dataset gsm8k-train \
  --input_jsonl /mnt/PK/dataset/gsm8k/main/train/data.jsonl
```


### Step 3: Recover answers from reasoning outputs

```bash
python recover_answer.py \
  --input_jsonl /mnt/PK/Inside-out/reasoning/32B-SFT/metamath/data/math/metamath-owmfilter/metamath.jsonl \
  --output_jsonl /mnt/PK/Inside-out/reasoning/32B-SFT/metamath/data/math/metamath-owmfilter/metamath_recovered.jsonl
```


### Step 4: Split dataset for probing

```bash
python split_for_probing.py \
  --input_jsonl /mnt/PK/Inside-out/7B/candidate_answer_set_prompt1/final/AH_rebuttal_new.jsonl \
  --train_ratio 0.25
```


## 📂 Script Descriptions

### 1. `ca_mcq.py`

Generates candidate answer sets for **factual (MCQ) datasets**.

* Input: JSONL file containing MCQ questions
* Output: Candidate answer set generated by the model. 
        Saved at: Inside-out/{model_size}/candidate_answer_set_prompt{prompt_id}


### 2. `ca_reasoning.py`

Generates candidate answer sets for **reasoning datasets**.

* Supports datasets such as GSM8K
* Output: Produces reasoning-based outputs. 
        Saved at: Inside-out/reasoning/{model_size}


### 3. `recover_answer.py`

Recovers final answers from reasoning outputs.

* If an answer already exists → use it
* If empty → extract the **last number** from the reasoning


### 4. `split_for_probing.py`

Splits dataset for probing-based evaluation.

* Creates:

  * `_probing` dataset (for training linear classifier)
  * `_labeling` dataset (for evaluation)
